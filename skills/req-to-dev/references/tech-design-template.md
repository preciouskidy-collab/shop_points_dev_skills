# 技术设计文档模板

本文档定义后端技术设计文档的标准结构，所有技术方案均应按此模板生成。

## 文档结构

1. **需求概述** — 背景、目标、范围
2. **功能点拆解** — 模块划分与职责
3. **系统架构设计** — 模块关系、技术架构、部署架构
4. **API 接口设计** — 接口总览与详情
5. **数据库设计** — 表结构、索引、ER 关系
6. **关键技术选型** — 决策点与理由
7. **风险与注意事项** — 已知风险及应对

## 各章节要求

### 需求概述

- 背景需说明业务驱动因素
- 目标需具体可量化
- 范围需明确边界（包含和不包含）

### 功能点拆解

- 按业务实体或功能域拆分模块
- 每个模块明确职责边界
- 列出功能点清单，标注优先级

### API 接口设计

- 遵循 RESTful 规范
- 路径格式：`/api/v1/{resource}`
- 请求/响应使用 JSON
- 统一响应结构：`{code, message, data}`

### 数据库设计

- 每张表必须包含：id, created_at, updated_at, is_deleted
- 字段命名使用 snake_case
- 必须明确索引设计（主键索引、唯一索引、普通索引）
- 遵循阿里巴巴 MySQL 规范

### 技术选型

- 每个决策点必须说明选择理由
- 列出备选方案及排除原因
